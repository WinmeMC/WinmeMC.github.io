响应式设计的核心逻辑

1. 流式布局

使用相对单位（%, vw, vh）而不是固定像素，让元素根据容器大小自动调整。

/* 固定布局（不响应式） */
.container {
    width: 960px; /* 固定宽度 */
}

/* 流式布局（响应式） */
.container {
    width: 100%;
    max-width: 1200px;
    margin: 0 auto;
}

2. 媒体查询

根据设备特性（如屏幕宽度）应用不同的CSS规则。

/* 基础样式（移动优先） */
.element {
    width: 100%;
}

/* 平板设备（768px及以上） */
@media (min-width: 768px) {
    .element {
        width: 50%;
    }
}

/* 桌面设备（992px及以上） */
@media (min-width: 992px) {
    .element {
        width: 33.33%;
    }
}

3. 响应式图片

确保图片在不同设备上都能正确显示，不破坏布局。

<!-- 响应式图片基本用法 -->
<img src="image.jpg" 
     alt="描述文字"
     style="width: 100%; height: auto;">

<!-- 响应式图片高级用法 -->
<img src="small.jpg"
     srcset="small.jpg 500w, medium.jpg 1000w, large.jpg 1500w"
     sizes="(max-width: 600px) 100vw, 50vw"
     alt="响应式图片">

Viewport 元标签 - 必须设置！

Viewport 是响应式设计的基石，它告诉浏览器如何控制页面的尺寸和缩放。

<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <!-- 必须设置的 viewport 标签 -->
    <meta name="viewport" 
          content="width=device-width, initial-scale=1.0">
    
    <!-- 其他属性说明 -->
    <!-- width=device-width: 页面宽度等于设备宽度 -->
    <!-- initial-scale=1.0: 初始缩放级别为1 -->
    <!-- maximum-scale=1.0: 禁止用户缩放（不推荐） -->
    <!-- user-scalable=no: 禁止用户缩放（不推荐，影响可访问性） -->
</head>

媒体查询详解

常用断点

业界标准断点（基于主流设备尺寸）

/* 超小设备（手机，小于576px）*/
/* 移动优先：先写手机样式 */

/* 小型设备（竖屏平板，≥576px） */
@media (min-width: 576px) { 
    /* 样式 */ 
}

/* 中型设备（横屏平板，≥768px） */
@media (min-width: 768px) { 
    /* 样式 */ 
}

/* 大型设备（笔记本/台式机，≥992px） */
@media (min-width: 992px) { 
    /* 样式 */ 
}

/* 超大型设备（大屏幕，≥1200px） */
@media (min-width: 1200px) { 
    /* 样式 */ 
}

媒体类型和特性

除了屏幕宽度，还可以根据其他特性应用样式

/* 仅针对屏幕设备 */
@media screen { ... }

/* 打印样式 */
@media print { 
    .no-print { display: none; } 
}

/* 横屏设备 */
@media (orientation: landscape) { ... }

/* 竖屏设备 */
@media (orientation: portrait) { ... }

/* 高分辨率屏幕（视网膜屏） */
@media (-webkit-min-device-pixel-ratio: 2), 
       (min-resolution: 192dpi) { ... }

/* 深色模式 */
@media (prefers-color-scheme: dark) { 
    body { background: #000; color: #fff; } 
}

复杂媒体查询

使用逻辑运算符组合多个条件

/* 与运算：同时满足两个条件 */
@media (min-width: 768px) and (max-width: 1024px) { 
    /* 仅适用于768px到1024px之间 */ 
}

/* 或运算：满足任意一个条件 */
@media (max-width: 600px), (orientation: landscape) { 
    /* 宽度≤600px 或 横屏设备 */ 
}

/* 非运算：排除某些条件 */
@media not (max-width: 600px) { 
    /* 宽度>600px的设备 */ 
}

/* 组合查询 */
@media screen and (min-width: 768px) and 
       (max-width: 1024px) and 
       (orientation: landscape) { 
    /* 平板横屏样式 */ 
}

现代响应式布局技术

1. Flexbox 弹性盒子

一维布局模型，非常适合组件和小规模布局

Flex 项目 1
Flex 项目 2 (flex: 2)
Flex 项目 3

调整窗口大小查看弹性效果

/* Flexbox 容器 */
.container {
    display: flex;
    flex-wrap: wrap; /* 允许换行 */
    gap: 1rem; /* 项目间距 */
    justify-content: space-between; /* 主轴对齐 */
    align-items: center; /* 交叉轴对齐 */
}

/* Flexbox 项目 */
.item {
    flex: 1 1 300px; /* flex-grow | flex-shrink | flex-basis */
    /* 解释：可伸展 | 可收缩 | 基础宽度300px */
}

/* 响应式Flexbox */
@media (max-width: 768px) {
    .container {
        flex-direction: column;
    }
    .item {
        flex: 1 1 auto;
    }
}

2. CSS Grid 网格布局

二维布局系统，适合页面级布局

Grid 项目 1
Grid 项目 2
Grid 项目 3
Grid 项目 4

网格会自动调整列数

/* CSS Grid 容器 */
.container {
    display: grid;
    
    /* 自动适应列数，每列最小200px，最大等分 */
    grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
    
    /* 或固定列数 */
    /* grid-template-columns: repeat(3, 1fr); */
    
    gap: 1rem; /* 行列间距 */
}

/* 响应式Grid */
.container {
    /* 默认：1列（手机） */
    grid-template-columns: 1fr;
}

@media (min-width: 768px) {
    .container {
        /* 平板：2列 */
        grid-template-columns: repeat(2, 1fr);
    }
}

@media (min-width: 1024px) {
    .container {
        /* 桌面：3列 */
        grid-template-columns: repeat(3, 1fr);
    }
}

响应式设计实战示例

响应式卡片组件

一个完整的响应式卡片组件，包含所有最佳实践

<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>响应式卡片</title>
    <style>
        /* 移动优先：基础样式（手机） */
        .card-container {
            display: grid;
            gap: 1rem;
            padding: 1rem;
        }
        
        .card {
            background: white;
            border-radius: 12px;
            overflow: hidden;
            box-shadow: 0 4px 6px rgba(0,0,0,0.1);
            transition: transform 0.3s ease;
        }
        
        .card:hover {
            transform: translateY(-5px);
        }
        
        .card img {
            width: 100%;
            height: 200px;
            object-fit: cover;
        }
        
        .card-content {
            padding: 1.5rem;
        }
        
        /* 平板：2列布局 */
        @media (min-width: 768px) {
            .card-container {
                grid-template-columns: repeat(2, 1fr);
            }
        }
        
        /* 桌面：3列布局 */
        @media (min-width: 1024px) {
            .card-container {
                grid-template-columns: repeat(3, 1fr);
            }
        }
        
        /* 大桌面：4列布局 */
        @media (min-width: 1280px) {
            .card-container {
                grid-template-columns: repeat(4, 1fr);
            }
        }
        
        /* 打印样式 */
        @media print {
            .card {
                break-inside: avoid; /* 防止分页打断卡片 */
                box-shadow: none;
                border: 1px solid #ddd;
            }
        }
    </style>
</head>
